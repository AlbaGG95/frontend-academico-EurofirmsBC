Box model y Position

Aquí separo caja y posicion para no mezclar conceptos.

Esta ficha me sirve para distinguir cuando un problema es de espacio, de tamano o de posicion dentro del flujo.

Ficha de estudio

Lo que quiero recordar antes de tocar el layout

Objetivo

Separar espacio, tamano y posicion

Si se que parte del problema pertenece a la caja y cual a la posicion, resuelvo mucho antes.

Error común

Tocar `position` cuando el fallo era de caja

A veces el layout esta mal porque hay margen, padding o ancho incorrecto, no porque falte posicionamiento.

Regla

Primero reviso box model

Antes de usar `absolute` o `fixed`, me interesa comprobar si el elemento ya esta mal dimensionado.

Box model

Qué hace cada parte

Espacio exterior

`margin`

Separa el elemento de lo que tiene alrededor.

Borde

`border`

Es el limite visible del elemento.

Espacio interior

`padding`

Da aire al contenido por dentro del elemento.

Tamano

`width` y `height`

Definen el tamano del contenido si no cambio el modelo de caja.

Position

Comportamientos que quiero distinguir

Flujo normal

`static`

Es el comportamiento por defecto.

Ajuste local

`relative`

Mueve el elemento sin sacarlo del flujo normal.

Fuera del flujo

`absolute`

Se posiciona respecto al ancestro posicionado más cercano.

Pegado a ventana

`fixed`

Se queda fijo respecto a la ventana del navegador.

Mixto

`sticky`

Se comporta normal hasta que alcanza una posicion concreta al hacer scroll.
