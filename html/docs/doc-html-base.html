HTML

Aquí dejo mi base de estructura y semántica.

Esta ficha me sirve para recordar las etiquetas que más voy a usar y la forma en la que quiero ordenar una página desde el principio.

Objetivo
Orden

Si la estructura esta clara desde el HTML, luego el CSS se vuelve mucho más fácil de manejar.

Ficha de estudio

Lo que quiero tener claro antes de escribir HTML

Para que esta página me resulte útil de verdad, dejo siempre el mismo esquema: objetivo, conceptos clave, errores y ejemplo.

Objetivo

Dar una estructura clara al contenido

Si el HTML esta bien planteado, luego me cuesta menos aplicar CSS y entender que parte de la página estoy tocando.

Criterio

Usar semántica cuando tenga sentido

Intento no tirar de `div` por defecto si una etiqueta como `header`, `nav`, `section` o `article` encaja mejor.

Error común

Montar la estructura sin pensar

Si empiezo a poner contenedores sin criterio, luego me cuesta mucho más encontrar el sitio correcto para estilos y clases.

Conceptos clave

Etiquetas que quiero usar con sentido

Cabecera

`header` y `nav`

`header` me sirve para la cabecera general o la cabecera de una sección. `nav` lo uso cuando el contenido principal son enlaces de navegación.

Contenido

`main`, `section` y `article`

`main` envuelve el contenido principal de la página. `section` me viene bien para bloques tematicos y `article` para piezas con sentido propio.

Cierre

`footer`

Lo uso para pies de página o de bloque cuando necesito información de cierre, enlaces secundarios o creditos.

Ejemplo mínimo

Estructura base que puedo reutilizar

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>Mi página</title> </head> <body> <header></header> <main> <section></section> </main> <footer></footer> </body> </html>
Si puedo empezar con una estructura limpia, luego tardo menos en colocar clases y estilos.
