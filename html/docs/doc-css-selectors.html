Selectores CSS

Aquí dejo lo que quiero recordar sobre selectores.

Esta ficha me ayuda a escribir reglas con más criterio y a no crear selectores raros que luego sean dificiles de mantener.

Ficha de estudio

Lo que quiero tener claro

Objetivo

Apuntar al elemento correcto

Si el selector esta bien pensado, el CSS es más corto, más claro y más fácil de corregir.

Error común

Hacer selectores demasiado largos

Si necesito encadenar demasiadas partes para llegar a un elemento, probablemente el HTML o las clases no estan bien planteados.

Regla

Primero clases claras

Prefiero nombres de clase con sentido antes que selectores complejos para corregir estructura mala.

Tipos

Selectores que más me conviene dominar

Clase

`.clase`

La uso para estilos reutilizables y para mantener el CSS modular.

.card { border-radius: 18px; }
ID

`#id`

Solo lo uso si necesito algo verdaderamente único en la página.

#hero { min-height: 100vh; }
Descendiente

`.padre .hijo`

Me sirve cuando quiero afectar a un hijo dentro de un contexto concreto.

.menu a { color: white; }
Hijo directo

`.padre > .hijo`

Lo uso cuando solo quiero afectar a los hijos inmediatos.

.menu > li { list-style: none; }
Pseudoclase

`:hover`, `:first-child`, `:last-child`

Sirven para estados o situaciones concretas sin tocar el HTML.

a:hover { opacity: 0.8; }
