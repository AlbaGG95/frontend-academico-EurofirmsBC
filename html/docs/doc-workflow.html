Metodología

Guía para documentar tareas nuevas y tareas continuas.

Esta ficha define cómo registrar un encargo, qué información debe quedar clara al cerrar una sesión y cuándo tiene sentido continuar una tarea existente en lugar de abrir un archivo nuevo.

Uso principal
Seguimiento

Sirve para mantener el hilo de trabajo entre sesiones y dejar el proyecto listo para retomarlo sin fricción.

Objetivo
Claridad

Una persona externa debe poder entender el estado de la tarea con una lectura rápida de la ficha.

Plantilla base

Información mínima que debería tener cada encargo

Si esta estructura se mantiene estable, cualquier práctica puede crecer sin perder orden ni trazabilidad.

Contexto

Qué se pidió y en qué módulo encaja

Describe el enunciado, la fecha, la materia del bootcamp y el objetivo concreto del trabajo.

Ejecución

Qué se ha construido

Resume los archivos implicados, la solución técnica aplicada y las decisiones importantes tomadas durante la sesión.

Verificación

Qué se ha comprobado

Indica si se ha revisado la estructura, la navegación, el diseño responsive, el foco visible o cualquier criterio relevante.

Continuidad

Qué queda para la siguiente sesión

Debe quedar claro si la tarea se cierra o si sigue abierta, junto con el próximo paso recomendado.

Regla práctica

Cuándo continuar una tarea y cuándo crear otra nueva

El criterio no debería depender del nombre del archivo, sino del alcance real del encargo.

Continuar

Misma tarea, nueva fase

Si el ejercicio añade mejoras, nuevas pantallas o un refinamiento del enunciado inicial, conviene mantener la misma práctica.

Crear nueva

Cambio real de alcance

Si el objetivo, la tecnología o la naturaleza del encargo cambian de forma clara, tiene sentido abrir un nuevo proyecto o ficha.

Registrar

Explicar por qué se tomó la decisión

Sea continuidad o proyecto nuevo, conviene dejar una nota breve que justifique la organización elegida.

Checklist

Puntos de revisión antes de cerrar una sesión

Estructura
  • El objetivo de la sesión está descrito.
  • Los archivos trabajados están identificados.
  • La solución aplicada se entiende sin contexto adicional.
Accesibilidad
  • La navegación principal sigue siendo clara.
  • Los enlaces tienen texto reconocible.
  • El foco visible y la jerarquía visual se mantienen.
Continuidad
  • Se indica si la tarea queda cerrada o sigue abierta.
  • Hay una nota de siguiente paso para la próxima sesión.
  • No se duplica trabajo que ya existía en otra ficha.

Bitácora breve para cada sesión de trabajo

Proyecto o práctica: Fecha: Estado: nueva / en progreso / cerrada Objetivo de la sesión: Archivos trabajados: Cambios realizados: Verificación aplicada: Bloqueos o dudas: Pendiente para la próxima sesión:
